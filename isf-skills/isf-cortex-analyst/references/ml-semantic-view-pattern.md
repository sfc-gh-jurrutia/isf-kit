# ML Explainability Semantic View Pattern

Template for a semantic view over the ML schema generated by `isf-data-architecture`. This enables natural-language queries about model performance, feature importance, and prediction explanations.

## When to Use

Create this view when:
- The solution includes ML notebooks that export to the ML schema
- The Cortex Agent should answer "why" questions about model predictions
- Users need to explore model performance without writing SQL

## YAML Template

```yaml
name: ml_insights_model
description: "ML model performance and explainability. Query feature importance, model metrics, calibration data, and parameter sensitivity."

custom_instructions: |
  ML TERMINOLOGY:
  - SHAP importance: How much each feature contributes to predictions (higher = more important)
  - PDP (Partial Dependence): How changing one feature affects the prediction while holding others constant
  - Calibration: Whether predicted probabilities match actual outcomes (important for decision thresholds)
  - Feature interaction: When two features together have a different effect than each alone

  QUERY ROUTING:
  - For "which features matter" → use feature_importance table
  - For "how accurate" or "performance" → use model_metrics table
  - For "how does X affect predictions" → use partial_dependence table
  - For "why did the model predict X" → use prediction_explanations table

tables:
  - name: feature_importance
    base_table:
      database: "{DATABASE}"
      schema: ML
      table: GLOBAL_FEATURE_IMPORTANCE
    primary_key:
      columns: [IMPORTANCE_ID]
    dimensions:
      - name: model_name
        expr: MODEL_NAME
        data_type: TEXT
        description: "Name of the ML model"
      - name: feature_name
        expr: FEATURE_NAME
        data_type: TEXT
        description: "Input feature name"
      - name: feature_direction
        expr: FEATURE_DIRECTION
        data_type: TEXT
        description: "Whether feature pushes prediction up (positive), down (negative), or varies (mixed)"
        sample_values: ["positive", "negative", "mixed"]
    facts:
      - name: shap_importance
        expr: SHAP_IMPORTANCE
        data_type: NUMBER
        description: "Mean absolute SHAP value — higher means more important"
      - name: importance_rank
        expr: IMPORTANCE_RANK
        data_type: NUMBER
        description: "Rank of this feature (1 = most important)"
    time_dimensions:
      - name: computed_at
        expr: COMPUTED_AT
        data_type: TIMESTAMP_NTZ

  - name: model_metrics
    base_table:
      database: "{DATABASE}"
      schema: ML
      table: MODEL_METRICS
    primary_key:
      columns: [METRIC_ID]
    dimensions:
      - name: model_name
        expr: MODEL_NAME
        data_type: TEXT
      - name: metric_name
        expr: METRIC_NAME
        data_type: TEXT
        description: "Performance metric type"
        sample_values: ["accuracy", "precision", "recall", "f1", "r2", "mae", "rmse"]
      - name: metric_context
        expr: METRIC_CONTEXT
        data_type: TEXT
        description: "Evaluation context"
        sample_values: ["train", "test", "validation"]
    facts:
      - name: metric_value
        expr: METRIC_VALUE
        data_type: NUMBER
      - name: sample_count
        expr: SAMPLE_COUNT
        data_type: NUMBER
    time_dimensions:
      - name: evaluated_at
        expr: EVALUATED_AT
        data_type: TIMESTAMP_NTZ

  - name: partial_dependence
    base_table:
      database: "{DATABASE}"
      schema: ML
      table: PARTIAL_DEPENDENCE_CURVES
    primary_key:
      columns: [PDP_ID]
    dimensions:
      - name: model_name
        expr: MODEL_NAME
        data_type: TEXT
      - name: feature_name
        expr: FEATURE_NAME
        data_type: TEXT
    facts:
      - name: feature_value
        expr: FEATURE_VALUE
        data_type: NUMBER
      - name: predicted_value
        expr: PREDICTED_VALUE
        data_type: NUMBER
      - name: lower_bound
        expr: LOWER_BOUND
        data_type: NUMBER
      - name: upper_bound
        expr: UPPER_BOUND
        data_type: NUMBER

  - name: calibration
    base_table:
      database: "{DATABASE}"
      schema: ML
      table: CALIBRATION_CURVES
    primary_key:
      columns: [CALIBRATION_ID]
    dimensions:
      - name: model_name
        expr: MODEL_NAME
        data_type: TEXT
    facts:
      - name: predicted_prob_bin
        expr: PREDICTED_PROB_BIN
        data_type: NUMBER
      - name: actual_frequency
        expr: ACTUAL_FREQUENCY
        data_type: NUMBER
      - name: bin_count
        expr: BIN_COUNT
        data_type: NUMBER

verified_queries:
  - name: top_features
    question: "What are the most important features for the {model_name} model?"
    use_as_onboarding_question: true
    sql: |
      SELECT feature_name, shap_importance, importance_rank, feature_direction
      FROM __feature_importance
      WHERE model_name = '{model_name}'
      ORDER BY importance_rank
      LIMIT 10

  - name: model_accuracy
    question: "How accurate is the {model_name} model?"
    use_as_onboarding_question: true
    sql: |
      SELECT metric_name, metric_value, metric_context
      FROM __model_metrics
      WHERE model_name = '{model_name}'
      ORDER BY metric_name

  - name: feature_sensitivity
    question: "How does {feature_name} affect predictions in {model_name}?"
    sql: |
      SELECT feature_value, predicted_value, lower_bound, upper_bound
      FROM __partial_dependence
      WHERE model_name = '{model_name}' AND feature_name = '{feature_name}'
      ORDER BY feature_value
```

Replace `{DATABASE}`, `{model_name}`, and `{feature_name}` with project-specific values from `isf-context.md`.
