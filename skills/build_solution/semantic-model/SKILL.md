# Semantic Model Generator

> Auto-generate Cortex Analyst YAML from existing Snowflake tables

## When to Load

- From `build_solution/SKILL.md` to generate semantic-model.yaml
- Standalone: `/speckit.semantic-model` for existing tables

## Prerequisites

- Snowflake connection configured
- Target table exists with data

## Workflow

### Step 1: Select Source Table

**Ask** user for table reference:

```
Generate semantic model from which table?

Enter fully qualified name: {DATABASE}.{SCHEMA}.{TABLE}

Or select from recent tables in current context.

[Enter table] [Browse schemas]
```

**⚠️ MANDATORY STOPPING POINT**: Wait for response.

**Log**: Record `configuration` — append to `specs/{solution}/decision-log.jsonl`: step "Semantic Model, Step 1: Source Table", value_selected (table reference), rationale.

### Step 2: Discover Schema

**Query** table structure:

```sql
SELECT 
    COLUMN_NAME,
    DATA_TYPE,
    IS_NULLABLE,
    COMMENT
FROM {database}.INFORMATION_SCHEMA.COLUMNS
WHERE TABLE_SCHEMA = '{schema}'
  AND TABLE_NAME = '{table}'
ORDER BY ORDINAL_POSITION;
```

**Query** sample values for categorization:

```sql
SELECT * FROM {database}.{schema}.{table} LIMIT 100;
```

### Step 3: Auto-Classify Columns

**Classify** each column as dimension or measure:

| Data Type | Default Classification | Override Triggers |
|-----------|----------------------|-------------------|
| VARCHAR, STRING | Dimension | - |
| NUMBER, FLOAT | Measure | If <20 distinct values → Dimension |
| DATE, TIMESTAMP | Time Dimension | - |
| BOOLEAN | Dimension | - |
| VARIANT, OBJECT | Skip | - |

**Present** classification for review:

```
## Column Classification

| Column | Type | Classification | Sample Values |
|--------|------|----------------|---------------|
| CUSTOMER_ID | NUMBER | Dimension (PK) | 1, 2, 3... |
| CUSTOMER_NAME | VARCHAR | Dimension | "John", "Jane"... |
| REGION | VARCHAR | Dimension | "North", "South"... |
| ORDER_DATE | DATE | Time Dimension | 2024-01-15... |
| REVENUE | NUMBER | Measure (SUM) | 150.00, 200.50... |
| QUANTITY | NUMBER | Measure (SUM) | 5, 10, 3... |

Detected time column: ORDER_DATE
Suggested primary key: CUSTOMER_ID

[Accept] [Modify classifications]
```

**⚠️ MANDATORY STOPPING POINT**: Wait for confirmation.

**Log**: Record `architecture_choice` — append to `specs/{solution}/decision-log.jsonl`: step "Semantic Model, Step 3: Column Classification", value_selected (Accept/Modify classifications), alternatives, rationale.

### Step 4: Configure Measures

**For each measure**, define aggregation:

```
Configure measures:

REVENUE:
  - Default aggregation: [SUM/AVG/COUNT/MIN/MAX]
  - Create additional metrics? (e.g., AVG_REVENUE)

QUANTITY:
  - Default aggregation: [SUM/AVG/COUNT/MIN/MAX]

[Configure] [Use defaults (SUM)]
```

### Step 5: Generate YAML

**Generate** semantic model YAML:

```yaml
# Auto-generated semantic model for {TABLE}
# Generated by /speckit.semantic-model

name: {table_name_lowercase}
description: "Semantic model for {TABLE} analysis"

tables:
  - name: {TABLE}
    description: "{table_description}"
    base_table:
      database: {DATABASE}
      schema: {SCHEMA}
      table: {TABLE}

    dimensions:
      - name: customer_id
        description: "Unique customer identifier"
        expr: CUSTOMER_ID
        data_type: NUMBER
        unique: true
        
      - name: region
        description: "Geographic region"
        expr: REGION
        data_type: VARCHAR
        sample_values:
          - "North"
          - "South"
          - "East"
          - "West"

    time_dimensions:
      - name: order_date
        description: "Date of order"
        expr: ORDER_DATE
        data_type: DATE

    measures:
      - name: total_revenue
        description: "Sum of revenue"
        expr: SUM(REVENUE)
        data_type: NUMBER
        default_aggregation: sum
        
      - name: total_quantity
        description: "Sum of quantity ordered"
        expr: SUM(QUANTITY)
        data_type: NUMBER
        default_aggregation: sum
        
      - name: order_count
        description: "Count of orders"
        expr: COUNT(*)
        data_type: NUMBER
        default_aggregation: count
```

### Step 6: Validate and Save

**Validate** YAML syntax and references:

```
Validating semantic model...

✓ YAML syntax valid
✓ All columns exist in source table
✓ Data types match
✓ Aggregations valid

Save to:
1. specs/{solution}/semantic-model.yaml
2. Snowflake stage @{database}.{schema}.SEMANTIC_MODELS/
3. Custom path

[1/2/3]
```

**⚠️ MANDATORY STOPPING POINT**: Wait for save location.

**Log**: Record `configuration` — append to `specs/{solution}/decision-log.jsonl`: step "Semantic Model, Step 6: Save Location", value_selected (Local project/Snowflake stage/Custom path), alternatives, rationale.

## Sample Questions Generator

**Optionally generate** sample questions:

```yaml
# sample-questions.yaml
questions:
  - question: "What is the total revenue by region?"
    expected_sql: |
      SELECT REGION, SUM(REVENUE) as TOTAL_REVENUE
      FROM {TABLE}
      GROUP BY REGION
      
  - question: "Show me the top 10 customers by revenue"
    expected_sql: |
      SELECT CUSTOMER_ID, SUM(REVENUE) as TOTAL_REVENUE
      FROM {TABLE}
      GROUP BY CUSTOMER_ID
      ORDER BY TOTAL_REVENUE DESC
      LIMIT 10
```

- ✋ After table selection (Step 1)
- ✋ After column classification (Step 3)
- ✋ Before saving (Step 6)

## Tips for Better Models

- **Descriptive names**: Use business terms, not column names
- **Rich descriptions**: Help Cortex understand context
- **Sample values**: Include for categorical dimensions
- **Multiple measures**: Create common aggregations (SUM, AVG, COUNT)
