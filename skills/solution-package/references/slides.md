# Presentation Slides Guide

## Purpose

Creates a **focused executive narrative** for 30-45 minute customer meetings.

## Story Arc

| Phase | Slides | Content |
|-------|--------|---------|
| Situation | 1-2 | Industry context, customer position |
| Complication | 3 | Pain, gap in current Snowflake usage |
| Resolution | 4-9 | Vision, solution, proof, path forward |
| Action | 10 | Clear next step |

## Slide Structure (7-10 slides)

| Slide | Purpose |
|-------|---------|
| 1 | Title, audience, and promise |
| 2 | Industry and customer context |
| 3 | Persona pain and current-state gap |
| 4 | Future-state vision and outcomes |
| 5 | Solution overview |
| 6 | Solution architecture & data journey |
| 7 | Persona journey & experience (optional) |
| 8 | Proof: customer story and value |
| 9 | Adoption path & value realization |
| 10 | Call to action & next conversation |

## Design Principles

### Full-Sentence Titles
State the insight, not the topic:
- ❌ "Architecture Overview"
- ✅ "Our unified data platform eliminates 80% of integration overhead"

### One Dominant Visual
Each slide has ONE main visual element:
- Diagram OR chart OR image
- Not multiple competing elements

### Answer-First (Pyramid Style)
- Lead with the conclusion
- Support with evidence
- Details in appendix

### Customer-Anchored
- Use their system names
- Reference their challenges
- Tie to their initiatives

## Quality Checkpoint

- [ ] 8-9 core slides (fits 20-25 minute delivery)
- [ ] Each slide has full-sentence insight title
- [ ] Architecture diagram is legible in conference room
- [ ] Proof slide includes quantified outcomes
- [ ] CTA is specific and actionable
